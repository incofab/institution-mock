Core Relationships

Institution
  has many Users through InstitutionUser
  has many ExamContent, Course, Grade, Student, Event, ExamActivation, Funding, GatewayPayment

ExamContent
  has many Course

Course
  belongs to Institution
  belongs to ExamContent
  has many CourseSession

CourseSession
  belongs to Course
  has many Question, Instruction, Passage

Event
  belongs to Institution
  belongs to ExternalContent
  has many EventCourse
  has many Exam
  has many ExamActivation

Exam
  belongs to Event
  belongs to Student
  has many ExamCourse
  belongs to ExamActivation

ExamActivation
  has many Exam

Funding
  belongs to Institution
  belongs to User
  morphs to nullable fundable source

GatewayPayment
  belongs to Institution
  belongs to User
  morph one Funding

Important Migrations

The main tables are created by migrations for institutions, institution users, external contents, events, exam contents, courses, course sessions, instructions, questions, passages, grades, students, event courses, exams, exam courses, and exam activations. The users table includes a nullable role column; the institutions table also includes licenses and license_cost; institution_users includes a role column defaulting to admin; successful license funding is stored in fundings, while hosted checkout lifecycle state is stored in gateway_payments.

Internal vs External Event Content

Internal events use database-backed EventCourse rows and eager-load courseSession.course, questions, instructions, and passages. External events use external_content_id; selected event courses are stored in the event's external_event_courses JSON field and rehydrated into EventCourse and CourseSession objects by model accessors.

Deletion Rules

  • Institutions cannot be deleted if they have courses or events.
  • Exam content cannot be deleted if it has courses.
  • Courses cannot be deleted if they have course sessions.
  • Course sessions cannot be deleted if they have questions.
  • Events cannot be deleted if they already have exams.
  • Grade deletion currently deletes its students before deleting the grade.
