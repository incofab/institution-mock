Purpose

CCD is the content authoring area. It manages exam content groups, courses, course sessions, instructions, passages, and questions. The routes use the same institution prefix and access middleware as the main institution area, with route names prefixed by institutions.ccd..

Exam Content

ExamContentController handles /exam-contents. An exam content record groups courses under a named exam body or content set.

  • index paginates all ExamContent records.
  • store and update validate exam_name, optional description, and optional fullname.
  • store creates through $institution->examContents().
  • destroy aborts when the content already has courses.

Courses

CourseController manages subjects/courses. Courses can be filtered by exam content id using /courses/index/{examContent?}.

FieldValidation
exam_content_idNullable integer that must exist in exam_contents.
course_codeRequired string.
course_titleOptional string.
descriptionOptional string.
orderOptional string used for ordering in event course lists.

Course deletion is blocked when the course has sessions.

Course Sessions

CourseSessionController is mounted as a nested resource at courses/{course}/course-sessions. A course session represents a year/session/category of questions for a course.

  • index loads sessions for the course with questions_count.
  • store creates a unique session per course using firstOrCreate.
  • update validates and saves session, optional category, and optional general_instructions.
  • destroy aborts if questions exist for that session.

Instructions and Passages

Instructions and passages are session-level supporting content. Both controllers render list/edit pages and store content under the current course session with the current institution id.

The route spelling currently uses couse-sessions in the index paths.

Questions

QuestionController stores questions under a course session. Question numbers are generated by checking the latest existing question_no and adding one for the create form.

FieldValidation
question_noRequired integer.
questionRequired string up to 60000 chars.
option_a, option_bRequired options.
option_c, option_d, option_eOptional options.
answerRequired and must be one of A, B, C, D, or E.
answer_metaOptional explanation/metadata.

Store uses updateOrCreate on question_no within the session. Bulk uploads use UploadSessionQuestionsRequest and the same storage path.

Uploads and Export

  • questions/course-sessions/{courseSession}/upload displays and processes Excel question upload.
  • courses/{course}/upload-content accepts a ZIP file up to 20 MB and calls UploadCourseContent::run().
  • courses/{course}/export-content calls ExportCourse::run() and currently exits with Done.
  • questions/course-sessions/{courseSession}/image-upload stores TinyMCE images and returns JSON with location. The controller references S3-style storage and image optimization.
