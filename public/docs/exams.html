Exam Lifecycle

  1. Institution user creates content in CCD: exam content, courses, course sessions, and questions.
  2. Institution user creates an event and attaches selected course sessions as event courses.
  3. Institution user registers an exam for one student, a grade, or a multi-student payload.
  4. RegisterExam creates an exams row with a generated exam_no, pending status, event duration as time_remaining, and related exam_courses.
  5. Exam start calls StartExam, marks pending/paused exams active, sets start_time and end_time, syncs an .edr file, and returns sanitized startup data.
  6. Attempt updates are stored in the exam tracking file as a question-id keyed attempts map.
  7. Exam end/evaluation calls EndExam, compares attempts with answers, updates course and total scores, marks the exam ended, and syncs the tracking file.
  8. Institution users activate a single exam or all unactivated exams in an event. A single activation creates one exam_activations row with num_of_exams, links the activated exams through exam_activation_id, and deducts the required licenses balance.
  9. Results can be viewed by exam number only after the exam has ended and has been activated, or exported per event as XLSX.

Status Model

App\Enums\ExamStatus defines pending, active, paused, and ended.

  • Exam::markAsStarted() sets status to active, clears pause_time, and calculates end_time from event duration.
  • Exam::markAsPaused() sets status to paused and clears start/end times.
  • Exam::markAsEnded() sets status to ended, saves score totals, and stores attempts.
  • Exam::getTimeRemaining() returns remaining seconds based on end_time.

Exam Tracking Files

ExamHandler stores per-exam data in public/exams/exam_{examNo}.edr. The file is JSON despite the custom extension. It contains an exam object with event, student, score, status, and timestamps, plus an attempts object keyed by question id.

{
  'exam': {
    'event_id': 1,
    'student_id': 1,
    'num_of_questions': 40,
    'score': 31,
    'status': 'active',
    'start_time': '2026-04-11 09:00:00',
    'end_time': '2026-04-11 10:00:00'
  },
  'attempts': {
    '15': 'A',
    '16': 'D'
  }
}

ExamHandler::getContent() checks file existence and, when enabled, checks if the file status is ended or the current time is past end_time. calculateScoreFromFile() compares each question answer with the saved attempt.

Activation

ActivateExam handles activation in a database transaction. It locks the institution license balance, skips exams that already have exam_activation_id, checks that enough licenses exist, creates one exam_activations row for the activation batch, stores the activated exam count in num_of_exams, links the exams to that row, and updates institutions.licenses. Activating an event charges only unactivated exams.

Existing Exam Activation Command

php artisan exams:activate-existing {institutionCode?} credits each selected institution with exactly enough existing-exams funding licenses for its current unactivated exams, then activates those exams by event. When institutionCode is supplied, only that institution is processed; without it, every institution is processed. Already activated exams are skipped.

Result Pages

Public result viewing is mounted at GET exams/view-result/{examNo?}. Exam\ExamController@viewResult displays exams.view-result-form when no exam number is supplied, loads the exam with examCourses, student, event, and event.institution, blocks unfinished or unactivated exams, then renders exams.view-result.

Time Extension

ExtendExamTime accepts minutes, chooses the later of now or the existing end_time, adds the duration, sets status back to active, clears pause time, and syncs the exam file.

Result Export

DownloadResult builds an XLSX with student name, subject scores, raw score, percent-style score sum, and date. EventController@download blocks the download when any exam in the event is unactivated, then writes the spreadsheet temporarily to storage/app/public, downloads it, and deletes it after sending.
