Response Style

API controllers use base response helpers such as apiSuccessRes() and apiFailRes(), plus Laravel validation exceptions. API routes currently have no custom auth middleware except the sample /user and /rough Sanctum routes.

Exam API

EndpointControllerPurpose
POST /api/exam/startAPI\ExamController@startExamValidates exam_no, loads exam with event and student, runs StartExam::getExamStartupData(), and returns exam_track, prepared exam, remaining time, and base URL. When exam.restrict_unactivated_exam_access_before_start is enabled, unactivated exams fail before the exam file is synced.
ANY /api/{exam:exam_no}/end-examAPI\ExamController@endExamRoute binds an exam by exam_no, runs EndExam::endExam(), and returns success.
ANY /api/exam/pauseExam\ExamController@pauseExamLegacy/runtime route declared in API routes. The visible Exam\ExamController only defines result viewing; verify before building clients against this route.
ANY /api/exam/endExam\ExamController@endExamLegacy/runtime route declared in API routes. Confirm controller implementation before use.
ANY /api/exam/submitExam\ExamController@submitExamLegacy/runtime route declared in API routes. Confirm controller implementation before use.

Result Upload API

ANY /api/institutions/{institution}/exams/upload maps to API\ExamController@uploadEventResult. It accepts a batch of exam records and updates local exams and exam_courses.

{
  'exams': [
    {
      'id': 1,
      'time_remaining': 0,
      'score': 31,
      'status': 'ended',
      'num_of_questions': 40,
      'attempts': {'15': 'A'},
      'exam_courses': [
        {
          'course_session_id': 7,
          'course_code': 'MTH',
          'session': '2025',
          'score': 31,
          'status': 'ended',
          'num_of_questions': 40
        }
      ]
    }
  ]
}

The method wraps updates in a transaction. Each exam course is updated or created by exam_id and course_session_id, then selected exam fields are updated.

CCD API Notes

  • POST /institutions/{institution}/api/questions/course-sessions/{courseSession}/store is the named CCD question API route inside the CCD web group, not the API route file.
  • ANY /api/ccd/institution/{institution_id}/question/create/{sessionId} maps to QuestionController@apiCreate in routes/api.php. Verify implementation before use because the visible QuestionController defines storeApi but not apiCreate.
  • POST /institutions/{institution}/questions/course-sessions/{courseSession}/image-upload is a web/CCD endpoint for TinyMCE image uploads and returns {'location': '...'}.
