Access

The institution route group uses web, auth, and institution.user. VerifyInstitutionUser reads the route-bound institution, checks that the authenticated user has an institution_users row for the institution or is the configured admin, shares $institution with GET views, and merges institution_id into the request.

Dashboard

InstitutionController@index renders institutions.index with counts for students and events.

Events

Events are exam windows owned by an institution. Each event has a title, duration, status, and optionally an external_content_id. Internal events use local event_courses; external events store selected external course-session data as JSON in external_event_courses.

FeatureRouteImplementation

Create/updateGET/POST/PATCH /eventsValidates Event::ruleCreate(). Create loads local courses and external content options.
Suspend/unsuspendGET /events/{event}/suspend, /unsuspendSets status to suspended or active.
Evaluate eventGET /events/{event}/evaluateRuns EndExam::endEventExams() over all event exams.
Download resultsGET /events/{event}/downloadBuilds an XLSX with DownloadResult and returns a temporary download.
DeleteDELETE /events/{event}Aborts if exams exist, deletes event courses, then deletes the event.

Event Courses

Event courses connect an event to the course sessions students may write. Internal records are stored with eventCourses()->updateOrCreate(). External event courses are built from ExternalContent::makeEventCourse() and saved back to the event's external_event_courses JSON field. Exam registration validates selected course session ids against $event->getEventCourses().
